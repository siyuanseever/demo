## 当前阶段：Mac Catalyst 稳定化与数据闭环

当前实现是 **iOS App 的 Mac Catalyst 迁移版**，不是原生 macOS App。当前阶段先让共同逻辑稳定、可观测、可测试；长期按 N0-N5 迁移到原生 macOS target。

优先级：

1. **P0 自动化治理**：先确保四个 Agent 读取同一协议、不会重复执行或错误提交。
2. **Critical 内存稳定性**：关闭 `MAC-MEM-GROWTH-001`，防止进程再次增长到系统级内存压力。
3. **P0 性能与鲁棒性**：解决卡死、慢点击、主线程数据库和同步阻塞。
4. **P0 数据架构与自动同步**：Python 后端为权威源，Mac 沙盒 SQLite 为缓存，自动刷新替代手动必选步骤。
5. **P0 数据/UI 完整性**：记忆层级、详情、最近更新和关联日记完整可达。
6. **P1 心流 ↔ 夜谈轻互动**：单卡片、可选轻动作、即时正反馈、无打卡负担。

Web/Python 只做 Mac 所需的数据契约、智能能力和缺陷修复，不主动扩展无关 Web 功能。

## 已确认的技术边界

- 当前使用 Catalyst 作为稳定化基线；长期迁移原生 macOS，但不以未经取证的重写代替卡死修复。
- Python 后端 `data/app.db` 是既有数据权威源。
- Mac App 使用沙盒 Documents 中的独立 `app.db` 作为缓存和离线降级。
- App 不直接持续读写仓库数据库文件；通过本机 API 自动增量刷新。
- 正常使用不依赖手动同步按钮；按钮保留为故障恢复入口。
- 完全脱离 Python 后端是未来独立架构项目，需要单独决策，当前不同时维护两套权威业务逻辑。

## 验收标准

| 领域 | 完成条件 | 必需证据 |
|---|---|---|
| 自动化 | 运行时使用 v3；每个 schedule slot 仅一个 run；每个 task 仅执行一次 | state/index、slot_id、claim 记录 |
| Mac 构建 | Catalyst 构建并启动，关键页面可进入 | xcodebuild 命令、退出码、启动存活证据 |
| 稳定性 | 关键路径无可复现卡死；无超过 1 秒主线程停顿 | 数据规模、复现步骤、trace、前后对比 |
| 内存安全 | 无持续单调增长；重复操作后回落；自动场景不触发 2GB 保护 | resident/footprint 曲线、Allocations、memgraph |
| 发送链路 | 点击发送后 UI 保持响应，阶段可追踪，后端在线时出现同一 correlation ID | App 阶段日志、后端接收日志、heartbeat |
| 自动同步 | 启动、回前台、写入后、后端恢复触发去重刷新 | 触发矩阵、网络/日志、缓存与更新场景 |
| 离线降级 | 后端不可用时立即显示缓存，不阻塞，不丢失本地可见内容 | 离线和恢复场景 |
| 长期记忆 | 一级类别、二级类别、叶节点详情均可点击到达 | 字段矩阵和导航记录 |
| 最近更新 | 新增/更新记忆和日记在刷新后可见 | 后端写入 → 自动刷新 → UI 证据 |
| 会后总结 | 摘要及三篇关联日记可进入；缺失数据有降级状态 | 完整/缺失两类场景 |
| 心流单卡片 | 主界面/夜谈每次最多一张相关卡，可进入来源 | 选择、冷却、导航、空状态 |
| 轻量行动 | 每卡最多一个可选动作；点击即时正反馈并记录；忽略无后果 | 点击/不点击两类场景 |

无法取得必需证据时必须标记 `pending_manual_validation` 或 `blocked`，不得标记完成。

## 自动化执行约束

- 当前 P0 模式：PM 每天 1 次；Executor 每天 1 次主执行 + 1 次仅重试 slot；Checker/Fixer 每 6 小时错峰运行。
- 稳定 7 天后：Checker 降为每 12 小时，Fixer 降为每天 1 次。
- PM 每个日程槽最多生成一个可执行任务；其余只写 backlog 建议。
- PM 在 main 只读运行，只写 handoff，不修改或提交仓库文件；其他三个 Agent 固定使用 automation worktree。
- Executor 必须先原子 claim 任务；旧协议、多任务、重复任务、错误分支一律拒绝执行。
- Checker 每 6 小时复验新产品提交和证据，不重复消费同一 Executor/Fixer run。
- Fixer 只处理 Checker 已复现的产品缺陷，不承担 Roadmap 功能开发。
- Git 同步是确定性前置步骤，不得由 PM 作为产品任务派发。
- 平台方向、数据权威源、迁移、安全策略和删除数据必须由用户或本文件明确。
- worktree/branch 生命周期只归用户/bootstrap；四个 Agent 发现缺失时必须停止。

## 主要风险

| 风险 | 应对 |
|---|---|
| App 与 Python 同时写同一数据库 | 使用 API + 独立沙盒缓存，不共享活动数据库文件 |
| 自动刷新引发更多卡顿 | 缓存先显示、增量拉取、去重、取消、后台事务 |
| 同一自动化任务重复执行 | schedule slot + task claim + completed key 三层幂等 |
| PM 从代码片段推断产品方向 | 方向不在 Roadmap/plan 时只能 `needs_human` |
| 点击互动变成任务负担 | 每卡一个可选动作；不点击无提醒、无惩罚 |

# 心理学知识检索

## 数据来源

- `cards.json`：原有手工维护卡片，继续承载内容推荐关系。
- `content_cards.json`：书籍、练习、音乐和电影等关联素材。
- `knowledge_card.schema.json`：合并后的运行时卡片契约。
- `response_plan.schema.json`：从召回卡片生成对话回应计划的契约。
- `taxonomy.json`：观察维度、解释层、卡片角色与检索安全策略。
- `supplemental_cards.json`：外部补充资料经去重、证据审查和去污名化后的新增卡片。
- `curated_retrieval_overrides.json`：格式化知识库中的精选别名、触发句、医学鉴别和过度病理化风险；合并进现有卡片，不复制概念。
- `docs/心理学知识卡片与检索系统资料库.md`：用户资料库。`document_cards.py`
  在启动时把第 2–13 章和 H1–H7 转为运行时卡片，不复制正文。

资料库缺失时系统会降级为只加载原有卡片，不影响启动。

## 检索与回答链路

1. `KnowledgeRetriever` 合并手工卡、文档卡与审校后的补充卡。
2. 用户原文、路由器的 `knowledge_queries` 和必要的记忆关键词共同参与召回。
3. 检索同时使用词语匹配、场景短语、元数据和少量同义表达扩展。
4. 文档卡根据共享标签和所属领域生成最多 4 条 `related_cards`，形成可追踪概念图。
5. 每轮最多返回调用方要求的卡片数；当前深度回复为 3 张。
6. 注入模型的每张卡都包含概念类型、证据边界、替代解释和章节来源。
7. `retrieve_plan()` 生成状态抽取、主卡、替代解释、拒绝的过度解读、医学鉴别、回应策略和安全标记。
8. 个体化假设只有在触发句高度匹配或长期记忆提供支持时才进入候选，并在排序时降权。
9. Web 聊天气泡显示本轮实际召回的卡片，同时显示至少一个替代解释和必要的身体鉴别。

## 统一组织方式

`taxonomy.json` 把知识分成四个互不替代的层次：

1. **观察**：事件、情绪、身体、想法、冲动、行为、关系/工作情境和时间过程。
2. **解释层**：先查现实与医学，再看唤醒负荷、认知、关系、学习史和意义。
3. **认识论状态**：明确区分事实、用户解释、候选机制、工作假设和临床诊断。
4. **干预**：动作必须可选、低负担、可停止，并包含安全边界。

Gemini 补充材料中的“背侧迷走神经强制关机”仅作为多迷走神经理论的检索
别名；“武器化脆弱”被改写为可观察的不对称情绪劳动；“冷酷雇佣兵心态”
被改写为有边界的策略性投入。产品不保留对第三方人格、潜意识或恶意动机的
确定判断。

这不是完整的向量检索。后续引入 embedding 时，应保留当前词法召回作为可解释
基线，并在融合排序后继续执行数量上限、替代解释和非诊断约束。

## 安全约束

- `working_concept` 不是标准诊断术语。
- `personalized_hypothesis` 不是用户事实。
- `contested_theory` 必须展示科学争议提示。
- 身体相关卡片必须保留医学解释。
- 关系拒绝、职场剥削等现实可能性不能被心理学解释覆盖。
- 卡片只能帮助讨论“此刻可能发生了什么”，不能回答“用户是什么”。

# 记忆与心理知识卡片检索审计

日期：2026-07-06

## 结论

本次审计确认了四个可复现问题：

1. 记忆重要度实际范围为 1–5，但混合检索阈值是 7，重要记忆层永远为空。
2. 每次查询无条件加入三条最近记忆，导致同三条记忆在 250 次查询中各出现 250 次。
3. 中文整句只按标点切分，记忆关键词即使包含在原句里也可能无法匹配。
4. 知识卡正文与标题、标签、触发词使用近似相同的匹配权重，宽泛卡片容易长期占据头部。

已对上述问题实施最小修复。同样本复测后，记忆空结果率、覆盖率和头部集中度均有明显改善；知识卡覆盖有所增加，头部集中度有所下降。

## 数据与方法

- 权威数据源：`data/app.db` 的临时副本，未写入原数据库。
- 样本：最近 250 条用户消息，其中 105 条能够从后续助手消息取得规划器检索词。
- 库存：304 条 active 记忆，116 张知识卡。
- 输出限制：只保留聚合指标、分类和不透明 ID，不保存或输出对话、记忆、日记正文。
- 对照方法：审计器同时模拟修改前算法，并运行当前算法，确保前后使用相同消息与相同卡片库。

这里的“命中”只表示算法返回了对象，不代表该对象一定适合最终回复。频率审计能发现偏置和失联对象，但不能代替人工相关性标注。

## 记忆检索发现

### 头部偏置

修改前，最近三条记忆各命中 250/250 次，是确定性的算法偏置，不是它们与每次对话都相关。改为仅在相关检索为空时回退后，混合检索 Top 10 占比由 64.7% 降为 34.8%。

### 长尾与零命中

修改后仍有 131 条记忆在该样本中没有命中：

| 一级分类 | 从未命中 |
|---|---:|
| emotion_pattern | 43 |
| self_core | 34 |
| resource_support | 19 |
| trauma_shadow | 16 |
| goal_action | 7 |
| relationship_pattern | 6 |
| body_response | 3 |
| life_habit | 3 |

其中 importance=5 的记忆仍有 32 条未命中，importance=4 有 72 条未命中。不能因此直接删除：样本主题分布可能没有触发它们，也可能是关键词、摘要或分类质量不足。需要人工相关性样本才能区分“合理沉默”和“不可检索”。

### 合并候选召回

旧逻辑只查看同一级分类最近 30 条。当前 `emotion_pattern` 有 95 条、`self_core` 有 84 条，旧记忆即使高度相似也可能根本进不了合并判断。候选窗口已扩大到每类 200 条，足以覆盖当前规模；最终 create/merge/update/contradict/ignore 仍由现有合并判断处理。

数据库中没有完全相同的规范化正文副本，但这不能证明不存在语义重复。当前缺少稳定的语义重复标注集，因此未自动合并或删除任何历史记忆。

### 二级分类

旧的自由生成标签有大量近义变体。固定分类上线后，原先 304 条中有约 280 条会落入 `general`，看板虽然不再无限增类，但信息结构仍然失真。

本次增加了只读兼容映射，不改写原始数据库。当前落入 `general` 的记录降为 61 条。新写入与更新仍会被约束到固定分类。映射属于启发式规则，后续应抽查高频映射，而不是立即批量覆盖历史原值。

## 知识卡发现

修改前的头部卡片主要是宽泛工作概念：

- `psych-library-6-4` 表演性生存：76 次
- `psych-library-2-6` 复杂性威胁：71 次
- `psych-library-5-3` 自我冬眠：43 次

原因不是这些卡必然最重要，而是正文很长、覆盖词很多，任何规划词落入正文都会累计较高分。修改后标题、别名、标签和检索触发词成为强匹配，正文只作为弱匹配；工作概念和临床概念增加轻度降权；无强匹配时至少需要两个正文证据。

修改后前三张仍然是上述工作概念，但频次分别降为 53、58、37。说明偏置减弱但没有完全消失。当前 32 张未命中卡片按类型分布：

| 类型 | 从未命中 |
|---|---:|
| established_concept | 15 |
| personalized_hypothesis | 6 |
| therapeutic_framework | 5 |
| working_concept | 5 |
| clinical_concept | 1 |

不建议按“未命中”直接删卡。治疗框架、临床概念和个性化假设本来就应低频、谨慎触发。真正需要清理的是内容高度重叠、缺少独特触发条件，或无法改变回复策略的卡片。

历史消息元数据中记录过 31 张卡、共 266 次使用。早期库规模较小，因此历史使用率不能直接与当前 116 张卡比较。

## 已实施修改

- 记忆查询增加“存储关键词是否直接包含在中文原句中”的匹配。
- 混合检索改为相关优先，只有零相关结果才回退到重要/近期记忆。
- 重要度阈值由无效的 7 调整为 5。
- 合并候选由同类最近 30 条扩大为最多 200 条。
- 更新记忆时同样规范化二级分类。
- 为历史自由标签增加固定分类兼容映射，但不改写数据库。
- 知识卡检索区分强字段和弱正文，降低宽泛正文与工作概念的支配效应。
- 新增脱敏审计命令：

```bash
python3 -m app.evaluation.retrieval_audit \
  --db data/app.db \
  --sample-size 250 \
  --output /tmp/retrieval-audit.json
```

## 仍需继续的工作

1. 建立 40–60 条人工标注的检索金标集。每条至少标注“必须命中、可接受、必须排除”，分别评估 Precision@3、Recall@5 和不当心理化率。
2. 为知识卡增加用途策略，而不只是相关性分数。例如区分 `response_eligible`、`internal_check_only`、`explicit_trigger_only`；治疗框架和临床概念默认不作为回复主解释。
3. 给检索结果记录脱敏原因：命中的触发词、字段、分数和最终是否被回复采用。没有“返回后是否使用”，单看命中频率仍无法判断卡片价值。
4. 为记忆合并建立人工抽查集，覆盖同义重述、时间变化、矛盾、短期状态与长期模式。之后再考虑字符相似度或向量召回，避免自动误合并。
5. 抽查当前 61 条 `general` 以及新映射中的高频类别，确认映射准确后再决定是否做显式数据迁移。
